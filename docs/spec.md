# html2md 行为规格

基于 BDD 场景描述，定义当前 Web 版 `html2md` 的可观察行为。

## 书架页

### 场景：进入首页时看到书架
- **当** 用户访问首页 `/`
- **那么** 系统必须展示书架页而不是单文件转换页
- **那么** 系统必须展示现有书籍列表或空书架状态

### 场景：书架为空
- **当** 本地存储中不存在任何书籍
- **那么** 系统必须展示空书架提示
- **那么** 系统必须提供“新建书籍”入口

### 场景：创建书籍
- **当** 用户点击“新建书籍”并确认输入书名
- **那么** 系统必须创建一本书籍记录
- **那么** 系统必须跳转到 `/?bookId=<id>` 对应的工作台页面

### 场景：导入整本书
- **当** 用户在书架页选择一个 `book.json`
- **那么** 系统必须校验其是否为受支持的书籍备份格式
- **那么** 系统必须把其中的书籍与章节数据恢复到本地存储
- **那么** 系统必须为导入结果创建新的本地 id，避免覆盖现有书籍
- **那么** 系统必须在导入成功后进入该书的工作台

### 场景：删除书籍
- **当** 用户在书架上删除一本书
- **那么** 系统必须弹出确认提示
- **那么** 确认后必须删除该书及其全部章节

### 场景：书架按最近更新时间排序
- **当** 书架中存在多本书
- **那么** 最近更新的书必须优先显示

## 书籍工作台

### 场景：进入书籍页面
- **当** 用户访问 `/?bookId=<id>`
- **那么** 系统必须加载该书信息与章节列表
- **那么** 如果书存在章节，系统必须默认选中第一章

### 场景：书不存在
- **当** 用户访问一个不存在的 `bookId`
- **那么** 系统必须返回首页

### 场景：首页作为静态路由入口
- **当** 用户访问首页 `/`
- **那么** 系统必须能够在纯静态部署环境下运行
- **那么** 书架与工作台切换必须通过查询参数而不是动态路径段完成

### 场景：修改书名
- **当** 用户点击“修改书名”并确认新书名
- **那么** 系统必须更新书名
- **那么** 系统必须更新书籍的 `updatedAt`

### 场景：创建章节
- **当** 用户点击“新建”创建章节
- **那么** 系统必须追加一个新章节
- **那么** 新章节必须自动进入当前编辑状态

### 场景：重命名章节
- **当** 用户修改章节标题并确认
- **那么** 系统必须更新该章节标题

### 场景：删除章节
- **当** 用户删除某一章节
- **那么** 系统必须弹出确认提示
- **那么** 确认后必须删除该章节
- **那么** 系统必须自动选中相邻可用章节；如果没有章节则保持空状态

### 场景：调整章节顺序
- **当** 用户点击章节上移或下移
- **那么** 系统必须更新章节顺序
- **那么** 导出顺序必须与该顺序一致

### 场景：章节选中状态明确
- **当** 用户选中某个章节
- **那么** 当前章节必须在左侧列表中具备明显的高亮样式与状态标识

## HTML 编辑与 Markdown 预览

### 场景：编辑 HTML
- **当** 用户在编辑区输入 HTML
- **那么** 系统必须将输入内容保存为当前章节的 HTML 草稿

### 场景：上传 HTML 文件
- **当** 用户上传 `.html` 或 `.htm` 文件
- **那么** 系统必须读取其内容
- **那么** 系统必须用文件内容覆盖当前章节 HTML

### 场景：拖拽 HTML 文件
- **当** 用户把 `.html` 文件拖入编辑区
- **那么** 系统必须读取文件内容并写入当前章节

### 场景：打开链接采集弹窗
- **当** 用户点击编辑器顶部的“链接采集”
- **那么** 系统必须弹出独立的链接采集窗口
- **那么** 用户不应离开当前章节工作台

### 场景：通过正文提取导入内容
- **当** 用户在链接采集窗口输入公开网页链接并点击“提取正文”
- **那么** 系统必须请求服务端正文提取接口
- **那么** 若提取成功，系统必须先展示预览内容
- **那么** 只有在用户点击“导入正文”后，才允许覆盖当前章节 HTML

### 场景：通过 iframe 选取导入内容
- **当** 用户在链接采集窗口加载页面并点击“进入选取”
- **那么** 系统必须在同源 iframe 中启用 DOM 选取
- **那么** 用户点选元素后，系统必须先展示预览内容
- **那么** 只有在用户点击“导入正文”后，才允许覆盖当前章节 HTML

### 场景：选取后支持切换父级或子级
- **当** 用户已经在预览区看到某个选取结果
- **那么** 系统必须展示该结果的完整标签路径
- **那么** 系统必须允许用户切换到父级
- **那么** 如果存在更细一级的原始选取结果，系统也必须允许用户退回子级

### 场景：站点触发风控时明确报错
- **当** 目标站点返回验证页、风控壳页或其他不可采集页面
- **那么** 系统必须直接提示采集被安全验证或风控拦截
- **那么** 系统不能误报为普通 iframe 超时

### 场景：提取与选取共用统一预览区
- **当** 正文提取成功或片段选取成功
- **那么** 系统必须使用同一块预览区域展示结果
- **那么** 当预览存在时，不应继续展示空白 iframe 占位区

### 场景：链接采集提供简短页面说明
- **当** 用户打开链接采集窗口
- **那么** 页面内必须提供一段简短说明，指引默认操作顺序

### 场景：主要操作按钮支持悬浮提示
- **当** 用户将鼠标悬停在主要采集按钮上
- **那么** 系统必须展示对应操作的简短说明
- **那么** 禁用态按钮也应能展示提示

### 场景：自动保存章节
- **当** 当前章节 HTML 与持久化内容不同
- **那么** 系统必须在约 800ms 防抖后自动保存
- **那么** 保存状态必须从“未保存”切换为“保存中”，再切换为“已保存”

### 场景：切换章节前冲刷草稿
- **当** 用户在当前章节存在未保存改动时切换到另一章节
- **那么** 系统必须先尝试保存当前草稿

### 场景：保存时自动转换 Markdown
- **当** 章节 HTML 被保存
- **那么** 系统必须立即调用转换器生成对应 Markdown
- **那么** 生成结果必须写回章节记录

### 场景：Markdown 预览只读
- **当** 用户切换到预览模式
- **那么** 系统必须展示当前章节 HTML 转换后的 Markdown
- **那么** 用户不能直接编辑 Markdown 内容

### 场景：Markdown 预览保持阅读排版
- **当** 用户查看 Markdown 预览
- **那么** 系统必须保留标题、段落、列表、引用、表格与代码块的阅读型排版
- **那么** `ul`、`ol`、`li` 的项目符号与编号不能被隐藏

### 场景：预览区域独立滚动
- **当** Markdown 内容超过可视高度
- **那么** 预览内容区必须内部滚动
- **那么** 页面整体不应因预览内容增长而继续拉长

### 场景：复制当前 Markdown
- **当** 用户点击“复制 MD”
- **那么** 系统必须将当前 Markdown 写入剪贴板

### 场景：下载当前 Markdown
- **当** 用户点击“下载 .md”
- **那么** 系统必须下载当前章节 Markdown 文件
- **那么** 下载文件名必须与当前章节标题一致
- **那么** 当章节标题为空或包含非法文件名字符时，系统必须生成可用的安全文件名

## 导出整本书

### 场景：没有章节时不可导出
- **当** 当前书籍没有任何章节
- **那么** 导出整本书按钮必须不可用

### 场景：导出整本书
- **当** 当前书籍存在章节并点击“导出整本书”
- **那么** 系统必须先保存当前待提交草稿
- **那么** 系统必须生成一个 zip 文件并触发下载

### 场景：zip 包含章节 Markdown
- **当** 导出成功
- **那么** zip 内必须包含所有章节 `.md`
- **那么** 文件名必须按顺序号命名，例如 `01-章节名.md`

### 场景：zip 包含目录文件
- **当** 导出成功
- **那么** zip 内必须包含 `README.md`
- **那么** zip 内必须包含 `toc.json`

### 场景：zip 包含可恢复备份
- **当** 导出成功
- **那么** zip 内必须包含 `book.json`
- **那么** `book.json` 必须包含继续编辑所需的书籍元数据、章节顺序、HTML 与 Markdown 内容

### 场景：重复或非法章节名处理
- **当** 章节标题为空、重复或包含非法文件名字符
- **那么** 系统必须生成稳定且安全的文件名

## 转换规则

### 场景：启用 GFM 扩展
- **当** 输入 HTML 包含表格、删除线或任务列表
- **那么** 系统必须输出对应的 GFM Markdown 语法

### 场景：转换 details 与 summary
- **当** 输入 HTML 包含 `<details>` 且存在 `<summary>`
- **那么** 系统必须将 `<summary>` 输出为加粗标题
- **那么** 系统必须展开并递归转换其余内容

### 场景：忽略无法直接映射的容器标签
- **当** 输入 HTML 包含 `<div>` 或 `<span>` 等容器标签
- **那么** 系统必须保留其文本内容并丢弃原始标签

### 场景：移除样式与脚本
- **当** 输入 HTML 包含 `<style>`、`<script>`、`<noscript>` 或 SVG
- **那么** 系统必须在转换时移除这些内容

## 主题与视觉

### 场景：切换深色模式
- **当** 用户点击明暗主题切换按钮
- **那么** 系统必须切换页面到对应主题
- **那么** 系统必须把选择持久化到本地存储

### 场景：深色模式统一生效
- **当** 页面处于深色模式
- **那么** 书架、工作台、按钮、标签、编辑器、预览器与弹窗都必须使用深色样式
- **那么** 不能出现明显残留的浅色激活态或浅色背景块

### 场景：视觉风格
- **当** 用户使用系统
- **那么** 页面整体应保持蓝白主色、扁平化、低装饰、阅读产品风格
- **那么** 不应依赖大面积渐变或厚重阴影来制造层级
