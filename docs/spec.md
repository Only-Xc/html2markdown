# html2md 行为规格

基于 BDD 场景描述，定义 `html2md` CLI 的可观察行为。

## 命令行模式选择

### 场景：使用位置参数进入单文件模式
- **当** 用户执行 `html2md input.html`
- **那么** 系统必须按单文件模式读取 `input.html` 并输出转换结果

### 场景：位置参数与输入目录参数互斥
- **当** 用户执行 `html2md input.html -i ./input`
- **那么** 系统必须向 stderr 输出 `Cannot use positional argument and -i together`
- **那么** 进程必须以退出码 `1` 结束

## 单文件模式输出路径解析

### 场景：未指定输出路径时写到标准输出
- **当** 用户执行 `html2md input.html`
- **那么** 系统必须将 Markdown 内容写到 stdout

### 场景：输出为 Markdown 文件路径
- **当** 用户执行 `html2md input.html -o output.md`
- **那么** 系统必须创建或覆盖 `output.md`

### 场景：输出为已存在目录
- **当** 用户执行 `html2md input.html -o ./out-dir` 且 `./out-dir` 已存在且为目录
- **那么** 系统必须将结果写入 `./out-dir/input.md`

### 场景：输出为不存在且无扩展名的路径
- **当** 用户执行 `html2md input.html -o ./generated/docs`
- **那么** 系统必须将该路径视为目录并自动创建
- **那么** 系统必须将结果写入 `./generated/docs/input.md`

### 场景：输出为不存在且扩展名不是 .md 的路径
- **当** 用户执行 `html2md input.html -o output.txt`
- **那么** 系统必须将该路径视为文件并直接写入 `output.txt`

## HTML 转 Markdown 规则

### 场景：启用 GFM 扩展
- **当** 输入 HTML 包含表格、删除线或任务列表
- **那么** 系统必须输出对应的 GFM Markdown 语法

### 场景：转换 details 与 summary
- **当** 输入 HTML 包含 `<details>` 元素且存在 `<summary>` 子元素
- **那么** 系统必须将 `<summary>` 内容输出为加粗标题
- **那么** 系统必须递归转换 `<details>` 内其余内容并在标题后展开输出

### 场景：忽略无法直接映射的容器标签
- **当** 输入 HTML 包含 `<div>` 或 `<span>` 等无法直接映射的标签
- **那么** 系统必须保留其文本内容并丢弃原始标签

## 批量目录转换

### 场景：使用默认目录
- **当** 用户执行 `html2md`
- **那么** 系统必须将 `./input` 作为输入目录
- **那么** 系统必须将 `./output` 作为输出目录

### 场景：保留子目录结构
- **当** 输入目录中存在 `docs/a.html`
- **那么** 系统必须将输出写入 `output/docs/a.md`

### 场景：输入目录没有 HTML 文件
- **当** 输入目录中不存在任何 `.html` 文件
- **那么** 系统必须向 stderr 输出 `No HTML files found in <dir>`
- **那么** 进程必须以退出码 `0` 结束

### 场景：批量处理部分失败
- **当** 批量模式下某个文件读取、转换或写入失败
- **那么** 系统必须向 stderr 输出该文件的错误
- **那么** 系统必须继续处理其他文件
- **那么** 处理完成后必须向 stderr 输出 `Done: X succeeded, Y failed`
- **那么** 如果 `Y` 大于 `0`，进程必须以退出码 `1` 结束

### 场景：检测符号链接循环
- **当** 递归扫描跟随符号链接时遇到循环引用
- **那么** 系统必须向 stderr 输出警告并跳过该路径

## 错误处理与 I/O 约束

### 场景：输入文件或目录不存在
- **当** 用户提供的输入文件或目录不存在
- **那么** 系统必须向 stderr 输出错误信息
- **那么** 进程必须以退出码 `1` 结束

### 场景：批量输出路径是普通文件
- **当** 批量模式的输出路径已存在且为普通文件
- **那么** 系统必须向 stderr 输出错误
- **那么** 进程必须以退出码 `1` 结束

### 场景：批量进度输出到标准错误
- **当** 系统在批量模式下处理文件
- **那么** 系统必须向 stderr 输出形如 `[1/5] input/docs/a.html → output/docs/a.md` 的进度信息
